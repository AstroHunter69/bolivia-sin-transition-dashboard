# Renewable Transition Dashboard v0

Open `index.html` in a browser to explore topology candidates, resilience stress cases, explicit candidate new lines, and corridor loading.

The dashboard is self-contained and uses the data generated by the new-line topology search.

## Interactions

- Switch topology candidate and stress case.
- Topology candidates use compact codes in the selector; the selected topology panel explains each code.
- Toggle existing grid, reinforced lines, new lines, plants/batteries, and labels.
- Click map lines, buses, renewable plants, batteries, ranking rows, or stress bars; use the topology element dropdown for specific corridors.
- Inspect solar, wind, and battery placement by candidate.
- Use the hourly stress behavior chart under the map to see whether the selected stress is a short overload spike, a sustained loading problem, or mostly a thermal-dispatch tradeoff.
- Use the `?` button in the header for the dashboard guide and metric FAQ.
- Read the selected topology candidate panel for a plain-language explanation of what each topology represents.
- The main physical-layer view uses an OpenStreetMap basemap underneath the electrical grid overlays.
- Existing grid lines use a restrained neutral overlay so new, reinforced, and max-stressed corridors remain visually dominant.

The basemap and Leaflet library load from public CDNs, so the map background requires internet access. The dashboard data itself is embedded locally.
